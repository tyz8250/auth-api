# 認証とは何か

認証とは、アクセスしてきたユーザーが「本人であるか」を確認する仕組みのこと。

## 認証は「あなたは誰か」を確認する仕組み

例えば、ログイン画面でメールアドレスとパスワードを入力してログインできるようにする仕組み。

## 認証と認可の違い

認証は「識別されたユーザーが本人かどうかを確認する行為」を確認する仕組み。
認可は「あなたはどのリソースにアクセスできるか」を確認する仕組み。

## signup / login / me の役割

- POST /api/auth/signup
  - ユーザを登録する
  - パスワードを　hash化して保存する
  - 次回以降、認証を行う際の土台を作る

- POST /api/auth/login
  - ユーザの認証を行う
  - 入力された password と、DB に保存されている password_hash を照合する
  - 認証に成功した場合、トークンを発行する

- GET /api/auth/me
  - ログインしているユーザの情報を取得する
  - トークンを使って、ログインしているユーザの情報を取得する

## password と password_hash の違い

password はユーザーが作成して、ログイン時に入力するもの。
ただし DB には password そのものを保存しない。

DB には password を hash 化した password_hash を保存する。

password_hash はトークンではない。
ログイン時に、入力された password が正しいか確認するために使う。

token はログイン成功後に発行され、
その後のリクエストで「ログイン済みの本人です」と示すために使う。

## token / session が必要になる理由

token / session は、
その後のログインで使うというより、
ログインした後のリクエストで本人として識別するために使う。
